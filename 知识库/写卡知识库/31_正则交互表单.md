正则交互表单
来源: 包分配社会角色卡实践
用途: 在 AI 输出中嵌入交互式 HTML 表单，用户填写后一键发回聊天框

════════════════════════════════════════
一、核心思路
════════════════════════════════════════

在角色卡第一条消息（或 AI 输出的任何位置）放置一个触发标记，
酒馆正则脚本匹配该标记，用完整的 HTML 表单替换显示。
表单收集用户输入后，通过 triggerSlash('/send ...') 将内容直接发送到聊天框。

整个流程无需脚本（index.ts），纯靠角色卡的「正则」扩展字段即可实现。


════════════════════════════════════════
二、实现步骤
════════════════════════════════════════

步骤 1: 在第一条消息中放置触发标记

  在第一条消息末尾写入一个不常见的标记文本，例如：

    【国家分配对象需求表】

  标记要足够独特，避免被 AI 随意生成。


步骤 2: 编写正则脚本（角色卡的「正则」扩展字段）

  在 index.yaml 的 扩展字段.正则 中配置：

    - 正则名称: 你的表单名称
      查找表达式: /【国家分配对象需求表】/s
      文件: 正则/你的表单文件名
      来源:
        用户输入: false
        AI输出: true          ← 匹配 AI 的输出
      作用于:
        仅格式显示: true      ← 替换显示内容
        仅格式提示词: true    ← 也替换提示词中的内容

  文件内容为用 ```html ``` 包裹的完整 HTML 表单代码。


步骤 3: HTML 表单的关键结构

  表单需包含以下部分：

  1) 表单收集区
     - 各类 <input>/<select>/<textarea> 收集用户输入
     - 每个字段给一个 id，方便 JS 取值

  2) 数据生成函数
     - collectFormData(): 从各输入框取值，空值给默认（如"不限"）
     - generateRequestText(data): 将数据格式化为结构化纯文本

  3) 发送逻辑

     ┌─────────────────────────────────────────────┐
     │  发送到酒馆聊天框的核心调用：                    │
     │                                               │
     │  triggerSlash(`/send ${text}`);               │
     │                                               │
     │  这是酒馆助手提供的全局函数，                   │
     │  无需 import，在 iframe 中可直接调用。           │
     └─────────────────────────────────────────────┘

     完整的发送函数示例：

       function sendToSillyTavern(text) {
         return new Promise((resolve, reject) => {
           if (typeof triggerSlash === 'function') {
             try {
               triggerSlash(`/send ${text}`);
               resolve();
             } catch (error) {
               reject(error);
             }
           } else {
             reject(new Error('未找到 triggerSlash 接口'));
           }
         });
       }

  4) 复制到剪贴板（降级方案）

     如果 triggerSlash 不可用，至少要能复制到剪贴板：

       async function copyToClipboard(text) {
         if (navigator.clipboard && window.isSecureContext) {
           await navigator.clipboard.writeText(text);
         } else {
           // 降级：创建临时 textarea
           const ta = document.createElement('textarea');
           ta.value = text;
           ta.style.cssText = 'position:fixed;opacity:0';
           document.body.appendChild(ta);
           ta.select();
           document.execCommand('copy');
           document.body.removeChild(ta);
         }
       }

  5) 主提交函数

       async function submitForm() {
         const formData = collectFormData();
         const requestText = generateRequestText(formData);

         // 始终复制到剪贴板作为保底
         await copyToClipboard(requestText);

         // 尝试自动发送
         try {
           await sendToSillyTavern(requestText);
           // 显示成功界面
         } catch (error) {
           // 降级提示用户手动粘贴
         }
       }

════════════════════════════════════════
四、注意事项
════════════════════════════════════════

1. 触发标记要唯一
   不要用 AI 可能自发输出的文本作为标记，否则会在不期望的位置渲染表单。

2. 两个版本可共存
   - 「仅复制版」：只用 clipboard，用户手动粘贴
   - 「自动发送版」：用 triggerSlash('/send ...') 直接发送
   可同时配置两个正则，通过「启用」字段切换，方便调试。

3. triggerSlash 是全局函数
   在 iframe 内的 script 标签中可直接调用，无需 import。
   但要注意 iframe 是否能访问到 parent 的 triggerSlash——
   酒馆助手的正则替换场景下可以，独立页面不保证可用。

4. 导出时正则文件要用 ```html ``` 包裹
   正则文件内容应是被 ```html ``` 包裹的完整 HTML 代码块，
   tavern_sync 会自动处理。

5. 防止重复提交
   用 isSubmitting 标志位锁定按钮，避免用户连续点击。

6. 表单关闭时机
   提交成功后建议用 innerHTML 替换为成功提示，
   再用 setTimeout 延时隐藏/关闭，给用户反馈。

7. 移动端适配
   - input 的 font-size 至少 16px 防止 iOS 自动缩放
   - 用 -webkit-tap-highlight-color 优化点击反馈
   - 按钮区域 min-height: 44px 左右，方便触控
