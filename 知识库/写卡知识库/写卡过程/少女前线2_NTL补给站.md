# 少女前线2_NTL补给站 — 写卡过程记录

## 项目类型

纯文字多角色角色卡（`index.yaml` + 世界书 + 第一条消息，无 `.ts` 文件）。

## 需求演进

### 初始需求
用户想写一张《少女前线2：追放》的NTL角色卡——{{user}}给指挥官戴绿帽，多角色，每个人形有不同的NTL风格。

### 关键决策点

| 决策 | 选择 | 理由 |
|------|------|------|
| 角色数量 | 6位核心 + N位过客 | 用户要求6人全上，再加其他人形可登场 |
| {{user}}身份 | 黄区补给站负责人 | 后勤/非战斗人员，与传奇指挥官形成对比 |
| NTL风格差异化 | 每人不同 | 索米温柔沦陷、闪电记忆替代、春田成熟崩溃、格琳娜怨恨报复、可露凯理性崩坏、黛烟愧疚循环 |
| 色色基调 | 略带脏感的背德快感 | 用户明确"不要沉重现实，要色色"，要求加入脏玩法 |
| 多人性色 | 允许但不强制 | 默认一对一，但多人场景自然发生 |
| 图生提示词 | 全部添加含R18标签 | 用户主动要求 |

### 黛烟的特殊处理
- 用户明确要求黛烟登场，提及"雷蒙先生的瓜"
- 处理策略：不点名"雷蒙"，用"那段让指挥官不快的过往"作为背景锚点
- 黛烟的NTL定位因此确定为"愧疚循环型"——有前科的人形在{{user}}的接纳中越陷越深

### NSFW玩法设计
- 每人详细的偏好体位、偏好玩法、脏玩法、不喜好、标志性行为
- 核心原则：**"留下痕迹"**——气味标记、内射、体液玷污旧物、公开场合的秘密
- 脏玩法与角色性格绑定：春田可在照片前被做，黛烟不能提那个名字

### 第一条消息重写
- **初版**：105行文艺描写，春田+索米+黛烟三人同时登场，氛围铺垫过长
- **用户反馈**："没看懂，太文艺太长"
- **终版**：25行，只有春田一人登场，开门见山——锁门、暗示没人、围裙滑落、犹豫吧台还是楼上、脱到一半让{{user}}选

### 绿灯改蓝灯
- **初版**：6位角色绿灯（关键词触发），符合多角色卡规范
- **用户反馈**："只有常驻的蓝灯才能保证我的缓存"
- **终版**：全部改为蓝灯常驻。Token消耗增加，但缓存命中率100%

## 踩坑记录

### 1. avatar.png 缺失导致 bundle 失败
```
Error: Invalid .png file header
```
- 原因：`src/少女前线2_NTL补给站/头像.png` 不存在（或为空文件）
- 解决：准备一个有效的占位头像放入该路径
- 用户后续需自行替换为实际角色头像

### 2. 图生提示词世界书 YAML 缩进错误
```
All mapping items must start at the same column at line 1963
```
- 原因：追加新条目时，`文件:` 字段缩进错误，与上级不对齐
- 解决：检查并修复缩进，确保 `文件:` 与 `递归:` 下的其他字段同级
- **还发现并修复了一个历史bug**：原文件末尾"风花雪月_苏谛斯"条目的 `文件:` 字段错误地嵌套在 `递归:` 内部

### 3. 图生提示词 push 时的 schema 错误
```
无效输入 → 路径: ["条目"][113] → 输入: {..."递归":{..."文件":"风花雪月_苏谛斯"}}
```
- 原因：`文件:` 字段被解析为 `递归` 的子字段，导致整个世界书结构schema验证失败
- 解决：修复原条目的缩进后重新push

### 4. 第一条消息过长
- 初版105行文艺描写，三人登场，心理活动丰富
- 用户反馈"太长太文艺没看懂"
- 教训：色色卡的开场白要**快准狠**——快速建立场景、准确交代人物、直接暗示基调。20-40行为宜

## 关键设计经验

### NTL角色卡的核心公式
1. **被绿者的存在要真实但不沉重**——指挥官是"遥远但真实"的背景，不是负面情绪的源头
2. **{{user}}的优势要明确**——不是比指挥官更强，而是比指挥官**更近、更日常、更记得**
3. **每个人形的沦陷路径要差异化**——同样的"日久生情"要有不同的 flavor
4. **脏玩法是性格的延伸**——春田可在照片前被做（执念崩溃），黛烟不能提那个名字（创伤保护）

### 多人场景的触发设计
- 不预设固定多人组合，而是让AI根据"谁同时出现在补给站"自然发挥
- 核心规则：不竞争、不嫉妒，"你也觉得{{user}}很好对吧"的默契分享
- 春田+任何人：春田主导并照顾对方
- 格琳娜+任何人：嘴硬但行动上不排斥，甚至会竞争

### 图生提示词的R18标签策略
- NAI标签格式：`[图像标签]` / `英文名` / `固定外貌` / `默认服装` / `R18标签` / `[/图像标签]`
- R18标签包含：体位、精液位置、表情、服装状态
- 标签设计与角色NTL定位呼应：春田有`cum on face`（崩溃型），可露凯有`expressionless`（理性崩坏型）

## 给未来 Agent 的提示

1. **判断项目类型**：先看 `src/` 下有没有 `.ts` 文件。没有就是纯文字角色卡，只读通用部分
2. **NTL角色卡的核心**：被绿者的存在感 + {{user}}的差异化优势 + 每个角色的独特沦陷路径
3. **色色卡的节奏**：开场白要短（20-40行），直接建立场景和暧昧基调，不要长篇文艺描写
4. **用户说"看不懂"时**：大概率是信息密度不够，而不是信息不够。砍掉冗余环境描写，保留动作和对话
5. **绿灯 vs 蓝灯的权衡**：多角色卡规范是"速览蓝灯+详情绿灯"，但如果用户要求缓存命中率，尊重用户选择改蓝灯
6. **图生提示词世界书追加时**：注意YAML缩进！`文件:` 字段必须与 `递归:` 下的其他字段同级，不能嵌套进去
7. **avatar.png 是必需的**：bundle前确保头像文件存在且是有效的png，可以从其他项目复制占位
8. **Dirty talk 要分层设计**：每个人的接受底线不同，写在"不喜好"里保护角色不崩坏

### 第一条消息扩展为6条（对应6个角色开局）

**需求**：用户要求根据封面提示词的6个方案，分别写不同的第一条消息，让玩家可以在酒馆里切换开局角色。

  - 文件: 第一条消息/4   # 可露凯

**6条消息设计对照表**：

| # | 角色 | 场景 | 核心互动 | 留给AI的钩子 |
|:---:|:---:|:---:|:---:|:---|
| 0 | 春田 | 吧台→楼梯口 | 脱到一半，犹豫吧台还是楼上 | "你选" |
| 1 | 索米 | 休息区沙发 | {{user}}的手钻进衬衫，她弓背 | "明天再说" + 信纸在颤 |
| 2 | 闪电 | 维修台 | {{user}}手指侵入敏感接口 | "他没有这样过" |
| 3 | 格琳娜 | 吧台 | 喝醉后咬上来，解腰带 | "就在这里，我不想被当作正经事" |
| 4 | 可露凯 | 维修台 | {{user}}解开剩余拉链 | "维护还没完成""这是任务需要" |
| 5 | 黛烟 | 储物间 | {{user}}从后面掀起裙子 | "对不起……但是别停" |

**设计原则**：
- 每条控制在 **25-35行**
- 前3-5行建立场景，5-8行角色状态+互动开始，结尾留给AI接手
- 每条都有"角色专属标志性台词/动作"作为钩子，确保AI能立刻进入该角色的节奏

### 封面NAI提示词迭代

**第一轮**：写了一版"含蓄暧昧"的封面提示词，被用户反馈 **"不够色"**

**第二轮**：重新设计为 **6套R18封面方案**，每人一个专属构图：
- 春田：吧台趴着被后入，回头看镜头笑，嘴角精液
- 索米：沙发侧躺融化，信纸掉在地上
- 闪电：维修台面无表情被侵入接口
- 格琳娜：吧台醉后坐台面，啤酒混精液
- 可露凯：维护台坐着被口，手指攥白
- 黛烟：储物间背对镜头，哭着说对不起

**核心设计变化**：从"暗示性"改为"直白展示"——姿势、裸露、体液、表情全部明确写入正面提示词，负面提示词增加NSFW专用排除（`clothes on, covered nipples, solo, no sexual activity`等）

### Discord简介迭代

**初版**：包含技术信息、使用方式、封面预览表格——用户反馈DC不支持表格，且内部信息不需要公开

**第二版**：删除技术信息/使用方式/封面预览，表格改为列表格式

**终版**：第一行加 **"简单来说"** 直白概括——
> **简单来说：你在废土补给站给格里芬指挥官戴绿帽，6个性格各异的前部下会上门找你，日常互动自然滑向色色，AI会主动推剧情。**

教训：Discord简介第一行就要让路人秒懂这是什么东西，文艺slogan放后面。
