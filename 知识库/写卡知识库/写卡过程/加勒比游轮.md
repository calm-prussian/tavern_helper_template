# 加勒比游轮角色卡 — 写卡过程记录

## 项目类型

纯文字角色卡（`index.yaml` + 世界书，无 `.ts` 文件），参考模板为 `src/度假沙滩`。

## 概念演进

### 版本1：明星派对游轮
- 参考 `度假沙滩` 的"沙滩性爱派对"模板
- 保留 世界书/同伴/ 的多组 toggle 结构（IP角色组 + NSFW变体）
- 新增普通人NPC（调酒师、潜水教练、千金大小姐、摄影师）
- **被用户否定**：不要"同伴"概念，{{user}}独自登船，所有乘客都是明星

### 版本2：纯明星游轮
- 删除普通人NPC，所有乘客改为全球各时代明星
- 3组明星：好莱坞黄金时代、亚洲经典、欧美当代
- 每组3个明星，每人3个文件（角色速览/组/NSFW）
- **被用户否定**：人太少，且不需要分组结构，每人一个条目即可
- 用户追加需求：需要AI能动态引入更多人的机制

### 版本3：明星游轮 + 乘客池（精简）
- 每人一个独立 YAML 文件（角色 + NSFW 合并）
- 新增 `乘客池.yaml`（蓝灯常驻），告诉 AI 船上~25人，提供生成模板
- 删除亚洲和欧美组，只留好莱坞三人（梦露/赫本/凯莉）
- **被用户全新创意取代**：召唤机制

### 最终版：召唤游轮
- 核心机制：每个乘客可召唤一个来自任何作品的虚构角色
- 船上两种人：~25真实乘客 + ~25被召唤角色 = ~50人
- 一人一个召唤角色，不满意回客舱换
- 召唤角色完全实体化，自由交互，包括色色
- 三名好莱坞明星转为"乘客池中被召唤角色示例"

## 设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 角色卡类型 | 纯文字 | 用户偏好简单直接，不需要MVU变量追踪 |
| 明星角色激活方式 | 绿灯（关键词） | 提到才激活，不浪费 token |
| 乘客池激活方式 | 蓝灯（常驻） | 始终告诉 AI 船上人多，可自由生成 |
| 同伴结构 | 每人一个独立文件 | 比分组结构更灵活，方便增减 |
| 第一条消息停在哪里 | 闭眼召唤前 | 让 AI 来接"召唤出谁"，给用户控制权 |

## YAML 格式踩坑

### 1. 关键字格式
- ❌ 逗号分隔：`关键字: - 梦露,玛丽莲,Marilyn`
- ✅ 独立条目：`关键字:\n  - 梦露\n  - 玛丽莲\n  - Marilyn`

### 2. 缩进一致性
- `启用`、`激活策略`、`插入位置`、`递归`、`文件` 必须同级（4空格）
- `类型`、`关键字`、`顺序`、`不可被其他条目激活` 必须同级（6空格）
- 关键字列表项（8空格）
- 常见错误：`激活策略:` 多缩了一格，YAML 解析为 `启用` 的子映射而报错

### 3. 世界书文件的 YAML 内容格式
```yaml
---
条目名:
  子项:
    - 列表项
```
- 文件以 `---` 开头
- 文件名无需与条目名一致
- 在 `index.yaml` 中用 `文件:` 引用（不带 `.yaml` 后缀）

├── 头像.png                     # 需用户自行提供
├── 第一条消息/0.txt              # 登船→得知规则→进客舱
└── 世界书/
    ├── 游轮世界观.yaml           # 蓝灯(顺序1)，设定+写作要求
    ├── 天气海况.yaml             # 蓝灯(顺序11)
    ├── 游轮场景.yaml             # 蓝灯(顺序10)，5个区域感官描写
    ├── NSFW引导.yaml             # 蓝灯(顺序18)，AI主动性指令
    ├── 船上活动.yaml             # 蓝灯(顺序20)，非色色活动
    ├── NSFW活动.yaml             # 蓝灯(顺序25)，色色玩法

9个世界书条目全部蓝灯常驻——因为船上没有固定角色，所有内容都是背景设定，AI 需要始终知晓。

## 与度假沙滩模板的主要差异

| 维度 | 度假沙滩 | 加勒比游轮 |
|------|----------|------------|
| 同伴 | 多组IP角色（FF7/巫师3等） | 无固定同伴，乘客池动态生成 |
| 角色结构 | 每组3文件（角色速览/组/NSFW） | 不需要，由乘客池覆盖 |
| 人多感 | 固定几组 toggle | 蓝灯乘客池 + 召唤机制，AI自由发挥 |
| 空间 | 平面沙滩 | 多层游轮（顶层/中层/下层） |
| 登场方式 | 和同伴一起开车到沙滩 | 独自登船，发现召唤规则 |

## 给未来 Agent 的提示

1. **先读 AGENTS.md** 判断项目类型，纯文字角色卡不需要 MVU/前端/脚本相关内容
2. **参考现有卡片**（`src/度假沙滩`、`示例/角色卡示例`）理解文件格式和 YAML 结构
3. **YAML 缩进是坑**——确保 `激活策略` 和 `启用` 同级，否则 taver_sync 会报 "Nested mappings" 错误
4. **关键字是独立条目**，不是逗号分隔字符串
5. **世界书文件以 `---` 开头**，最外层是条目名称映射
6. **插入顺序有讲究**——世界观最早，场景/引导次之，具体内容最后
7. **提问先于实施**——概念改动大时，先和用户确认核心机制再写代码
8. **乘客池模式**是为 AI 提供"无限角色"的有效方法：给模板 + 示例池，让 AI 自行生成，而非预写每一个角色
