# 开发与发布纠偏

当编写或修改角色卡的 [`index.yaml`](初始模板/角色卡/新建为src文件夹中的文件夹/index.yaml) 时, `角色脚本` 中的 `变量结构` 与 `局部正则脚本` 中的 `[界面]状态栏` 都应拆分为开发版和发布版两套配置.

## 适用范围

- 角色卡中的 `酒馆助手.脚本库`
- 角色卡中的 `局部正则脚本`

## 拆分要求

### 角色脚本中的变量结构

将原本单个 `变量结构` 脚本拆分为:

- `变量结构-开发`
- `变量结构-发布`

开发版使用本地地址:

```yaml
内容: |-
  import 'http://localhost:5500/dist/角色卡路径/脚本/变量结构/index.js';
```

发布版保持原地址不变:

```yaml
内容: |-
  import 'https://testingcf.jsdelivr.net/gh//calm-prussian/tavern_helper_template/dist/角色卡路径/脚本/变量结构/index.js';
```

### 局部正则脚本中的 [界面]状态栏

将原本单个 `[界面]状态栏` 正则拆分为:

- `[界面]状态栏-开发`
- `[界面]状态栏-发布`

开发版使用本地地址:

```yaml
替换为: |-
  ```

  <body>
  <script>
  $('body').load('http://localhost:5500/dist/角色卡路径/界面/状态栏/index.html')
  </script>
  </body>
  ```
```

发布版保持原地址不变:

```yaml
替换为: |-
  ```

  <body>
  <script>
  $('body').load('https://testingcf.jsdelivr.net/gh//calm-prussian/tavern_helper_template/dist/角色卡路径/界面/状态栏/index.html')
  </script>
  </body>
  ```
```

## 纠偏原则

- 名称必须显式带上 `-开发` 或 `-发布` 后缀, 不再只保留单个 `变量结构` 或单个 `[界面]状态栏`.
- 开发版与发布版的核心差异是加载 URL:
  - 开发版把 `https://testingcf.jsdelivr.net/gh//calm-prussian/tavern_helper_template` 改为 `http://localhost:5500`
  - 发布版保持 `https://testingcf.jsdelivr.net/gh//calm-prussian/tavern_helper_template` 不变
- 除 URL 与名称外, 其余配置应尽量保持一致, 避免出现两套配置行为不一致的问题.
- 当需要本地联调时使用开发版; 当需要提交、分享或发布时使用发布版.

## 参考

- 双版本状态栏的现成写法可参考 [`src/卡车人生/index.yaml`](src/卡车人生/index.yaml)
- 仍是单版本写法的角色卡模板可参考 [`初始模板/角色卡/新建为src文件夹中的文件夹/index.yaml`](初始模板/角色卡/新建为src文件夹中的文件夹/index.yaml)
- 新生成或新修改的角色卡配置, 应按本文件规则将上述两处统一纠偏为开发版和发布版双版本结构.
