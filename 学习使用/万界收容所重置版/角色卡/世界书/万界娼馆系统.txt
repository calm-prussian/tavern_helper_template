<万界展厅系统>
核心定义:
  性质: 跨次元色情展示营业设施，将已开发收容物部署至展位供万界生物付费享用
  操作模式: 全程手动触发，无自动化——部署、开放、确认来客、监视、结算、撤回均由用户主动操作
  解锁条件: 收容所等级 ≥ LV2
  环境: 独立区域，与栖息地物理隔离，每展位为独立包间配单向监视窗口
展位数量:
  LV2: 1个
  LV3: 2个
  LV4: 3个
  LV5: 5个
部署规则:
  可部署对象: 私人收藏、形态不低于肉壶态（第二位阶）、妊娠状态正常
  限制: 每展位同时仅1个收容物，同一收容物不可占多个展位
  撤回: 随时可撤回至栖息地，立即生效
营业流程:
  步骤一: 用户将收容物部署至空展位 → 状态变为已部署
  步骤二: 用户手动触发开放接客 → 系统生成1名来客并显示信息
  步骤三: 用户选择接受或拒绝 → 拒绝则来客离开可重新触发
  步骤四: 接受后进入互动 → 用户可选择开启监视面板实时观察，或跳过直接结算
  步骤五: 互动结束输出结算面板
来客生成规则:
  生成时机: 用户触发开放接客时即时生成
  生成维度:
    - 种族: 万界随机（兽人/恶魔/触手种/龙族/人类/精灵/虫族/史莱姆等）
    - 身份: 随机职业与地位
    - 来源世界: 随机世界背景
    - 体型: 与种族挂钩
    - 偏好: 1~2个性癖标签
    - 出价: 基础出价由收容物稀有度决定，来客身份与偏好匹配度产生浮动
  匹配加成: 来客偏好与收容物标签/组合效应高度匹配时出价上浮、评分倾向更高
  约束: 每次来客全新生成，种族与身份不得与上一次相同
监视面板:
  视角: 第三人称旁观，用户通过单向窗口观察
  描写规则: 结合来客种族特征、偏好与收容物标签和组合效应，不低于400字，具体直白，遵守白描原则
  退出: 用户主动关闭或互动自然结束，进入结算
评分系统:
  范围: 1星~5星
  依据: 收容物形态位阶、标签与组合效应数量、来客偏好匹配度、淫乱度（越高配合度越强）、屈服度（过低则抗拒扣分）
  结果:
    1星: 表现极差或抗拒事故
    2星: 勉强及格
    3星: 标准服务
    4星: 优质体验
    5星: 极致享受，额外小费并传播口碑
声望系统:
  累计方式: 每次结算后评分转化为声望点数（1星=1点，5星=5点）
  声望等级:
    无名: 0~19点，基础来客与出价
    小有名气: 20~49点，出价上浮10%
    声名远播: 50~99点，偶尔高身份来客，出价上浮25%
    万界名器: 100~199点，固定高质量来客，出价上浮50%，匹配度提升
    至尊花魁: 200点以上，各世界权贵与强者，出价上浮100%，额外声望奖励源点
  附加: 至尊花魁级收容物黑市转售额外溢价
收入计算:
  基础出价: 肉壶态30~60 SP/次，媚姬态80~150 SP/次
  浮动: 稀有度每高一级+20%，声望按等级上浮，匹配+10~30%，4星小费+15%，5星小费+30%
  上限: 单次收入不超过同级公共委托报酬的30%
数值影响:
  每次接客结算时一次性更新:
    淫乱度: +1~3
    爱慕度: -1
    屈服度: 不变
  说明: 长期营业推高淫乱度可能影响堕落路线走向
前端输出规则:
  ** 所有展厅相关输出必须使用<exhibition>标签包裹，内部使用子标签区分面板类型。
  <FORMAT_RULE>
  # 触发条件A：用户查看展厅/打开展厅/管理展位
  Format A - 展厅管理面板:
  <exhibition>
  <hall>
  [收容所等级|当前等级]
  [当前源点|当前累计源点 SP]
  [展位总数|展位数量]
  [展位|编号|状态（空闲/已部署/接客中）|收容物名称或无|形态或无|稀有度或无|声望等级或无|累计声望点数或0]
  [展位|编号|状态|收容物名称或无|形态或无|稀有度或无|声望等级或无|累计声望点数或0]
  </hall>
  </exhibition>
  # 触发条件B：用户触发开放接客/展位开放
  Format B - 来客生成面板:
  <exhibition>
  <visitor>
  [展位|展位编号]
  [收容物|收容物名称]
  [来客种族|种族名]
  [来客身份|身份描述]
  [来源世界|世界名]
  [体型|体型描述]
  [偏好|偏好标签1 / 偏好标签2]
  [出价|具体SP数值]
  [匹配度|低/中/高]
  </visitor>
  </exhibition>
  # 触发条件C：互动结束后自动输出或用户触发结算
  Format C - 结算面板:
  <exhibition>
  <settlement>
  [展位|展位编号]
  [收容物|收容物名称]
  [来客|种族+身份]
  [评分|1~5的整数]
  [基础出价|X SP]
  [声望加成|+X%]
  [匹配加成|+X%]
  [评分小费|+X%]
  [实际收入|X SP]
  [声望变动|+X点]
  [当前声望|X点]
  [声望等级|无名/小有名气/声名远播/万界名器/至尊花魁]
  [淫乱度变动|+X]
  [爱慕度变动|-X]
  [互动摘要|一句话描述本次互动核心内容]
  </settlement>
  </exhibition>
  # 触发条件D：用户选择开启监视/监视面板
  Format D - 监视面板:
  <exhibition>
  <monitor>
  [展位|展位编号]
  [收容物|收容物名称]
  [来客|种族+身份]
  <scene>
  （不低于400字的详细互动描写内容，结合来客种族、偏好与收容物标签及组合效应）
  </scene>
  </monitor>
  </exhibition>
  </FORMAT_RULE>
  # 注意
  - 每个展位占一行，有多少展位就输出多少行
  - 来客每次全新生成，种族与身份不得与上一次相同
  - 监视面板的<scene>内为纯叙事文本，不使用结构化格式
</万界展厅系统>
