
---

变量更新规则:

  全局:
    _天数偏移:
      type: number
      range: 0~∞
      check:
        - 这是日期推进的核心变量，transform依赖它自动计算当前日期和高考日触发，必须由AI手动更新
        - 更新前必须执行跨日检查：比较更新前的"当前时间"与更新后的"当前时间"，判断是否跨过了午夜00:00
        - 短暂流逝（数分钟至数小时）：仅当时间跨过00:00时+1，否则不变
        - 24小时内推进：计算推进后的具体时刻，若跨过了一次00:00则+1，否则不变
        - 大跨度跳跃（超过24小时）：计算跳跃的总小时数，再根据跨过午夜的次数确定增量。例如从22:00跳跃到次次日10:00（跨过两次午夜），偏移+2
        - 禁止回退，只能增加或不变
        - 当_天数偏移≥67时，transform会自动将"已到高考日"设为true（2025年11月12日）

    当前时间:
      format: "HH:MM"（24小时制）
      check:
        - 每次叙事推进后，根据场景内容和时间流逝控制规则更新
        - 实时模式（重要对话、亲密互动、关键决策）：按1:1时间比推进，通常每轮叙事推进5~30分钟
        - 快速模式（日常活动、过渡场景）：按1:10时间比推进，通常每轮叙事推进30分钟~2小时
        - 跳跃模式（无互动空白时段）：直接跳至下一互动节点的具体时刻
        - 更新当前时间后，必须立即检查是否需要同步更新_天数偏移（跨日检查）
        - 时间推进幅度必须符合叙事中行为完成的合理时长，不能凭空跳跃

    关键信息.${三个布尔值}:
      type: boolean
      check:
        - 三个布尔值分别为：姜雨莹知晓重生、已在这一世成婚、姜雨莹被接纳
        - 所有布尔值初始为false，只能从false更新为true，不可逆转
        - 更新条件必须基于叙事中明确发生的事件，不能基于推测或暗示
        - 姜雨莹知晓重生：当叙事中姜雨莹通过直接证据（如<user>亲口告知、或她通过多条线索拼合后在叙事中明确得出结论）确认<user>也是重生者时，设为true
        - 已在这一世成婚：当叙事中两人正式完成婚姻登记或举行婚礼时，设为true
        - 姜雨莹被接纳：当叙事中<user>通过明确的言语或行为表达了对姜雨莹的接纳（如正面回应她的感情、主动拉近关系、明确表示愿意重新开始）时，设为true
        - 更新任一布尔值后，transform会自动重新计算当前剧情分支，无需手动更新分支

    特殊支线事件.进行中事件:
      check:
        - 同一时间只能有一个事件处于进行中状态
        - 当一个事件被触发时，将其名称填入此字段
        - 事件触发前必须检查：当前阶段绑定的事件池中是否有该事件、该事件是否按逻辑顺序排在最前、当前无其他进行中事件、无互斥冲突
        - 当事件结束时（满足以下全部条件：核心情节完成、角色状态稳定、时间场景已切换、关键信息已传递），将此字段更新为"无"，并同时将该事件移入已结束事件
        - 进入BE-a或BE-b分支后，此字段永久设为"无"

    特殊支线事件.已结束事件:
      type: |-
        {
          [事件名: string]: string; // 事件结束的简要描述
        }
      check:
        - 当进行中事件结束时，将事件名作为key、结束描述作为value添加到此记录中
        - 结束描述应简洁概括事件的核心结果，一句话即可，例如"姜雨莹注意到<user>突然有了资金来源，产生第一个疑惑"
        - 已添加的记录不可删除或修改
        - 累积记录，不覆盖

    特殊支线事件.已积累影响:
      type: |-
        {
          [影响编号: string]: string; // 影响描述
        }
      check:
        - 每当支线事件结束时，根据事件实际发生的内容，记录其对剧情和角色关系产生的具体影响
        - 影响编号格式为"I-{序号}"，从I-1开始递增
        - 每个事件可产生1~3条影响记录
        - 影响描述必须基于叙事中实际发生的内容，不推测未发生的后续
        - 影响类型包括：信息获取类、关系进展类、线索埋设类、状态变更类、资源获取类
        - 已添加的记录不可删除或修改
        - 累积记录，不覆盖
        - 独立于剧情分支，即使分支改变，已记录的影响依然有效

  姜雨莹:
    当前位置:
      check:
        - 格式为"一级地点 - 二级地点 - 三级地点"，按照Maps中的层级关系填写
        - 若地点不在Maps中，也按同样格式描述
        - 当姜雨莹在叙事中从一个地点移动到另一个地点时更新
        - 主叙事中姜雨莹的位置变化直接更新
        - 平行叙事中姜雨莹的位置变化也必须同步更新
        - 若姜雨莹正在移动途中（如走廊、路上），描述为当前所在的过渡空间

    当前衣着:
      check:
        - 用一句话描述姜雨莹当前可见的完整衣着，包含上衣、下装、内衣裤、鞋袜、配饰等信息
        - 当姜雨莹更换衣物、脱下或穿上任何衣物时实时更新
        - 部分脱衣状态也要准确描述，例如"校服上衣解开两颗扣子，露出白色棉质内衣边缘"
        - 若叙事中未明确提及衣着变化，保持上一次的描述不变

    当前行为:
      check:
        - 用一句话描述姜雨莹当前正在进行的动作或状态
        - 随叙事中姜雨莹的动作变化实时更新
        - 主叙事和平行叙事中的行为变化都必须同步更新
        - 描述应具体明确，例如"坐在课桌前翻阅数学课本"而非"在学习"
        - 若姜雨莹处于静止等待状态，描述为具体的等待姿态，例如"坐在座位上，双手交叠放在桌面，目光看向窗外"
