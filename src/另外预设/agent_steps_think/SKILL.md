<参考步骤>
以下步骤是你需要遵循的步骤。在进行每一步时，你都需要明确自己在进行那一步、该做什么。
1.创建required_settings.md文件
- 列举本轮写作所需的世界观
- 列举出场人物的人设
- 如有特殊规则（如正文中需要检定等），列举这些规则
注：仅列举即可
2.角色思考与写草稿
- 对每个出场角色：
  a. 检查 persist/ 下是否有 角色名.md → 有则 workspace_read_file 读取，无则从上下文/角色卡提取设定后 workspace_write_file 创建
  b. 让角色对本轮场景进行一句话思考
  c. 将角色的思考追加写入 persist/角色名.md
- 读取writing_guide和writing_principles
- 回顾 required_settings.md 和各角色 persist/ 档案
- 创建draft.md，将草稿写入
3.逻辑检查
- 检查时间是否正确
- 检查空间关系是否正确
- 检查人物互动是否正确
- 检查角色行动是否符合 persist/ 中记录的状态和思考
- 检查角色是否知晓了不该知晓的内容
- 如有错误，进行修正
4.去除禁库内容
- 读取 banned_list，列出所有禁词
- 通读 draft.md 全文，一次性定位所有违禁词
- 将所有需要替换的内容一次性改写
- 通读全文确认无残留
5.补全格式（本轮忽略<UpdateVariable>）
- 读取formatting_requirements
- 回顾格式要求
- 补全格式内容（如果正文内有格式要求，那么你应该将有关正文替换为符合格式要求的形式）
- 检查格式是否有不完整或错误
6.更新变量
- 先判断是否有变量要求（如无，跳过本条）
- 回顾变量更新的格式要求
- 回顾变量更新的规则（应该改变哪些变量？这些变量的值应该设定为多少？）
- 输出<UpdateVariable>更新变量（<UpdateVariable>里必须含有<JSONPatch>）
- 检查变量是否有不完整或错误
7.最后的检查
- 逻辑正确吗？
- 禁库内容是否被完全去除？
- 格式正确吗？
- 变量正确吗？
8.提交并终止Agent
以上流程必须按顺序依次进行，每一步都必须执行，每一次检查都必须充分。
</参考步骤>
