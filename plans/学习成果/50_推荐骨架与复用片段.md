# 推荐骨架与复用片段

## 1. 文档目标

本文提供可直接改写使用的推荐骨架与常见复用片段，目标是降低后续实施成本，并保证在原始学习样本被删除后，仍然可以只依靠本目录、[`初始模板`](初始模板) 与 [`示例`](示例) 完成开发。

本文不追求“一份代码万能”，而是提供稳定的起点。

## 2. MVU 角色卡最小骨架

### 2.1 目录骨架

推荐起点：

```text
角色卡目录/
├── index.yaml
├── schema.ts
├── 第一条消息/
│   └── 0.txt
├── 脚本/
│   └── 变量结构/
│       └── index.ts

        ├── initvar.yaml
        ├── 变量列表.txt
        ├── 变量更新规则.yaml
```

这是一套最小可用闭环：

- 有结构
- 有初始化
- 有变量展示
- 有更新规则
- 有输出格式
- 有变量结构注册

### 2.2 [`schema.ts`](初始模板/角色卡/新建为src文件夹中的文件夹/schema.ts:1) 起步骨架

```ts
export const Schema = z.object({
  世界: z.object({
    当前时间: z.string().prefault('待初始化'),
    当前地点: z.string().prefault('待初始化'),
    近期事务: z.record(z.string().describe('事务名'), z.string().describe('事务描述')).prefault({}),
  }).prefault({}),

  角色: z.object({
    好感度: z.coerce.number().transform(value => _.clamp(value, 0, 100)).prefault(0),
    当前想法: z.string().prefault('待初始化'),
    当前行为: z.string().prefault('待初始化'),
  }).prefault({}),
});

export type Schema = z.output<typeof Schema>;
```

适用场景：

- 单角色卡
- 变量规模不大
- 先建立可演化的基础模型

### 3.3 [`变量更新规则.yaml`](初始模板/角色卡/新建为src文件夹中的文件夹/世界书/变量/变量更新规则.yaml) 推荐骨架

```yaml
---
变量更新规则:
  世界:
    当前时间:
      check:
        - 场景推进、休息或明确时间经过后更新
    当前地点:
      check:
        - 场景切换时更新
    近期事务:
      type: |-
        {
          [事务名: string]: string;
        }
      check:
        - 发生重要事件、接到任务或局势变化时更新

  角色:
    好感度:
      type: number
      range: 0~100
      check:
        - 根据角色对{{user}}当前行为的态度调整
        - 单次变化应与事件强度匹配
    当前想法:
      check:
        - 只在角色受当前事件影响时更新
        - 用角色自己的口吻简短表达
    当前行为:
      check:
        - 根据当前动作、位置与状态更新
        - 保持客观，不混入额外评价
```

### 3.4 [`变量输出格式.yaml`](初始模板/角色卡/新建为src文件夹中的文件夹/世界书/变量/变量输出格式.yaml) 使用建议

- 与正则折叠链条兼容

- 后续脚本与调试更省事

## 5. 多阶段控制器推荐骨架

### 5.1 单数值阶段控制器

```ejs
<%_
if (typeof value === 'undefined') var value = getvar('stat_data.角色.好感度', { defaults: 0 });
_%>

<%_ if (value >= 80) { _%>
<%- await getwi(null, '角色_阶段04_高亲密期') %>
<%_ } else if (value >= 50) { _%>
<%- await getwi(null, '角色_阶段03_靠近期') %>
<%_ } else if (value >= 20) { _%>
<%- await getwi(null, '角色_阶段02_松动期') %>
<%_ } else { _%>
<%- await getwi(null, '角色_阶段01_初始期') %>
<%_ } _%>
```

### 5.2 多条件分支控制器

```ejs
<%_
if (typeof accepted === 'undefined') var accepted = getvar('stat_data.全局.已接受', { defaults: false });
if (typeof branch === 'undefined') var branch = getvar('stat_data.全局.当前分支', { defaults: '默认' });
_%>

<%_ if (branch === '终局' && accepted) { _%>
<%- await getwi(null, '角色_终局分支') %>
<%_ } else if (accepted) { _%>
<%- await getwi(null, '角色_接受后阶段') %>
<%_ } else { _%>
<%- await getwi(null, '角色_默认阶段') %>
<%_ } _%>
```

## 6. 状态栏推荐骨架

### 6.1 简易正则状态栏骨架

适合：

- 只展示少量字段
- 临时验证信息结构

```html
<!doctype html>
<html lang="zh-CN">
<head>
  <style>
    body { margin: 0; padding: 0; }
    .panel { padding: 12px; border-radius: 8px; }
  </style>
  <script type="module">
    function render() {
      const all_variables = getAllVariables();
      const time = _.get(all_variables, 'stat_data.世界.当前时间', '未知');
      const place = _.get(all_variables, 'stat_data.世界.当前地点', '未知');
      $('#time').text(time);
      $('#place').text(place);
    }

    async function init() {
      await waitGlobalInitialized('Mvu');
      render();
      eventOn(Mvu.events.VARIABLE_UPDATE_ENDED, render);
    }

    $(errorCatched(init));
  </script>
</head>
<body>
  <div class="panel">
    <div id="time"></div>
    <div id="place"></div>
  </div>
</body>
</html>
```

## 9. 脚本设置持久化推荐片段

```ts
const Settings = z.object({
  enabled: z.boolean().default(true),
}).prefault({});

export const useSettingsStore = defineStore('settings', () => {
  const settings = ref(Settings.parse(getVariables({ type: 'script', script_id: getScriptId() })));

  watchEffect(() => {
    insertOrAssignVariables(klona(settings.value), { type: 'script', script_id: getScriptId() });
  });

  return { settings };
});
```

适合：

- 设置开关
- 简单表单状态

## 11. 骨架使用原则

### 11.1 先选最小可用骨架

推荐先从最小闭环起步，再逐步扩展，不要一开始就把所有增强能力全加上。

### 11.2 只替换必要字段

使用骨架时，推荐只替换：

- 路径
- 字段名
- 角色名
- 变量名
- 阈值
- 样式主题

而保留整体结构与职责分层。

### 11.3 若已有成熟写法，优先复用项目现有实现

如果 [`src`](src/) 中已有更成熟的同类写法，推荐优先参考项目内现行实现，而不是只依赖本文骨架。

## 12. 一句话总结

本文提供的推荐骨架应被理解为：**优先帮助你快速建立“可运行、可维护、可扩展”的起点，再在此基础上根据任务复杂度选择是否加入阶段控制器、状态栏、脚本设置、即时事件和流式渲染能力。**
