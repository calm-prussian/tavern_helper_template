# 学习成果总纲与使用顺序

## 1. 文档定位

本目录用于沉淀对项目内三类材料的综合学习结果：

- 业务范式样本
- 项目骨架模板
- 工程实现示例

这些学习成果应当在原始学习样本被删除后，仍然可以独立使用。因此，本目录中的文档必须尽量直接给出结论、结构、模式与取舍原则，而不是依赖回看一次性样本。

## 2. 本目录的使用目标

后续在本仓库中编写角色卡、世界书、脚本、前端界面、流式楼层界面、MVU 变量结构时，推荐优先阅读本目录中的沉淀文档，再结合现有模板与示例落地实现。

推荐把本目录视为：

- 设计阶段的统一知识入口
- 实施阶段的结构与模式清单
- 删除旧学习素材后的长期参考资料

## 3. 使用顺序

推荐按以下顺序阅读与使用：

1. 先看 [`plans/学习成果/00_总纲与使用顺序.md`](plans/学习成果/00_总纲与使用顺序.md)
   - 明确本目录各文档分别解决什么问题
   - 明确后续任务该优先查哪一类沉淀
2. 再看 [`plans/学习成果/10_角色卡标准结构.md`](plans/学习成果/10_角色卡标准结构.md)
   - 确认目录骨架、入口文件、角色卡配置结构
3. 再看 [`plans/学习成果/20_MVU变量与世界书规范.md`](plans/学习成果/20_MVU变量与世界书规范.md)
   - 确认变量结构、初始化、更新规则、输出格式与世界书协作方式
4. 若任务涉及动态人设、阶段切换、即时事件，再看 [`plans/学习成果/30_阶段控制器与事件组织.md`](plans/学习成果/30_阶段控制器与事件组织.md)
5. 若任务涉及界面、脚本、状态栏或流式渲染，再看 [`plans/学习成果/40_前端界面脚本与状态栏模式.md`](plans/学习成果/40_前端界面脚本与状态栏模式.md)
6. 实际开写前，再看 [`plans/学习成果/50_推荐骨架与复用片段.md`](plans/学习成果/50_推荐骨架与复用片段.md)
   - 直接复用推荐骨架与常用片段
7. 遇到多种方案冲突时，再查 [`plans/学习成果/90_常见取舍与冲突处理.md`](plans/学习成果/90_常见取舍与冲突处理.md)

## 4. 三类参考来源的长期分工

虽然原始学习素材可能会被删除，但学习后的长期分工应保持清晰：

### 4.1 结构骨架

结构骨架长期以 [`初始模板`](初始模板) 为主参考。

重点包括：

- 角色卡目录应该怎么组织
- 哪些入口文件是最小可用集合
- 前端界面、脚本、流式楼层界面的最小挂载方式是什么
- 角色卡 [`index.yaml`](初始模板/角色卡/新建为src文件夹中的文件夹/index.yaml) 中最基础的变量区、正则区、脚本库区怎么放

### 4.2 工程实现

工程实现长期以 [`示例`](示例) 为主参考。

重点包括：

- 推荐的 [`Schema`](示例/角色卡示例/schema.ts:1) 写法
- 推荐的 [`defineMvuDataStore`](示例/角色卡示例/界面/状态栏/store.ts:1) 使用方式
- 推荐的 [`mountStreamingMessages`](示例/流式楼层界面示例/index.ts:1) 流式挂载方式
- 推荐的脚本设置持久化、Pinia 状态管理、Vue 组件组织方式

### 4.3 业务抽象

业务抽象不再依赖被删除的学习素材，而以本目录中的沉淀文档为主参考。

重点包括：

- 角色卡适合拆成哪些世界书层级
- 变量系统如何与叙事、事件、状态栏协作
- 多阶段控制器如何根据变量决定启用哪个阶段
- 何时适合用纯正则状态栏，何时应该改成 Vue 状态栏
- 何时用脚本、何时用前端界面、何时用流式楼层界面

## 5. 统一设计原则

### 5.1 业务与实现分离

应先确定业务结构，再选择实现方式。

也就是说，优先回答这些问题：

- 这是单角色卡、群像卡，还是系统型角色卡
- 需要静态世界书，还是动态阶段切换
- 需要 MVU 变量，还是普通脚本变量即可
- 需要展示型状态栏，还是交互型前端界面
- 需要普通渲染，还是流式渲染

之后再决定要不要使用：

- [`registerMvuSchema`](示例/角色卡示例/脚本/变量结构/index.ts:1)
- [`defineMvuDataStore`](示例/角色卡示例/界面/状态栏/store.ts:1)
- [`mountStreamingMessages`](示例/流式楼层界面示例/index.ts:1)
- 正则状态栏替换
- Vue 组件化状态栏

### 5.2 优先复用现成结构

当仓库中已有成熟写法时，推荐优先复用，而不是重新自造体系。

优先复用的对象包括：

- [`初始模板/角色卡/新建为src文件夹中的文件夹`](初始模板/角色卡/新建为src文件夹中的文件夹)
- [`初始模板/前端界面/新建为src文件夹中的文件夹`](初始模板/前端界面/新建为src文件夹中的文件夹)
- [`初始模板/流式楼层界面/新建为src文件夹中的文件夹`](初始模板/流式楼层界面/新建为src文件夹中的文件夹)
- [`示例/角色卡示例`](示例/角色卡示例)
- [`示例/脚本示例`](示例/脚本示例)
- [`示例/流式楼层界面示例`](示例/流式楼层界面示例)

### 5.3 推荐而非机械照搬

本目录中的内容是推荐模式，不是死板约束。

如果出现以下情况，可以偏离推荐模式：

- 当前题材需要特殊世界书组织方式
- 当前界面复杂度明显超出常规状态栏
- 当前变量模型与常见角色卡差异极大
- 当前任务更适合使用脚本直接处理，而不是建立完整前端界面

但即使偏离，也应尽量保证：

- 目录结构清楚
- 变量结构自洽
- 叙事规则和世界书条目相互支持
- 代码风格与仓库现有实践不冲突

- 不把关键结论只写成“参见某旧样本”
